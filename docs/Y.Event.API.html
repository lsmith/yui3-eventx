Class Y.Event.API

Augmentation class to add event related API methods to instances of the host class. Use via Y.augment(MyClass, Y.Event.API, true).

Constructor

Y.Event.API ( config )
Parameters:
config <Object> (optional) configuration object for this instance (see _initEvents for details)

Properties

_event - protected static {Object}

Default prototype for emitted event objects. Contains the following propeties and methods:

  • _prevent
  • _stop
  • preventDefault()
  • stopPropagation()
  • stopImmediatePropagation()
  • halt() - stopPropagation() + preventDefault()
  • detach()

defaultEvent - static {Object}

Default lifecycle methods for any unknown event being referenced by the API. This is the generic custom event behavior.

To define specific events (only needed if they require special behavior), you can describe them statically on the class in an _events property. Typically, you'll want to use the class's defaultEvent as the prototype of any special events. E.g.


. MyClass._events = {
.     events: {
.         // define special behavior for a 'foo' event on all instances
.         foo: Y.Object(Y.Event.API.defaultEvent, {
.             init: function (host, subscription, callback) { ... }
.             destroy: function (host, type) { ... }
.             // all other behaviors inherited from the defaultEvent
.         })
.     }
. };
. Y.augment(MyClass, Y.Event.API, true);

If defining new default behaviors for a class, the default event definition will need to implement the following methods:

  • subscribe(host, subscription_kernel, args*)
  • detach(host, subscription, args*)
  • fire(host, type, *)
  • filterSubs(subsArray, subscription_kernel, args*)

Return false from either to prevent the subscription or detach respectively.

The API also supports the following properties/methods:

  • init(host, subscription_kernel, args*)
  • allowDups (true|false) property
  • destroy(host, type)
  • test(type, args*) (function)

Each method is called passing the event definition as 'this', and the calling object as the first arg (except for filterSubs). You can include any more or methods or properties on an event definition, but the definition object should not manage state (e.g. fireOnce should not update a property on the event definition) unless it that state applies to all instances and derived event definitions.

See the descriptions for the individual eventDef methods and properties.

eventDef.allowDups - protected static {boolean}

Control whether duplicate subscriptions to this event should be allowed. If false, the API will use the event definition's filterSubs method to search the existing subscriptions for duplicates. If one is found, the subscription will be aborted before reaching the subscribe method.

TYPE_PATTERN - protected static {Regexp}

Pattern used to split detach category from event type. By default, It matches on("foo:bar",..) as category "foo" and type "bar", and on("baz",..) as category "" and type as "baz".
Default Value: /(?:([^:]+?):)?(.*)/

Methods

_initEvents

protected void _initEvents ( config )

Sets up the event collection, subscription collection, default event, and other important state on the instance for managing and dispatching events.

Accepts a config object with the following properties:

  • events - A collection of specific event definitions that override default behaviors for the augmented class.
  • defaultEvent - event definition to use as a fallback for references to events not specifically included in the events collection.
  • path - bubble path

If a config is not specified for the instance, it defaults from a static _events property on the class constructor. If a class constructor is not present, the events collection is left empty, allowing all events to fall back to the defaultEvent methods. If a defaultEvent is not identified, the global Y.Event.API.defaultEvent is used.

Parameters:
config <Object> (optional) configuration for this instance

_resolveBubblePath

protected static Array _resolveBubblePath ( root )

Flattens the bubble path for a given root instance. Used by the default fire definition, but available for any customized fire logic. Flattens using a breadth-first algo, so given the following bubble structure:

. (D)  (E)   (D)  (F)
.    \  |     |  /
.     (B)     (C)       bubble up to
.        \   /
.         (A)           bubbles up to

The resulting bubble path would be [A, B, C, D, E, F], and not [A, B, D, E, C, F] (depth-first). Also note duplicate targets are ignored. The first appearance in the bubble path wins.

Parameters:
root <Object> the origin of the event to bubble (A in the diagram)
Returns: Array
the ordered list of target instances

after

Object after ( type , arg* )

Subscribe to an event on this object. Subscribers in this "after" phase will NOT have access to prevent any default event behaviors (if the event permits prevention), but will also not be executed unless the default behavior executes.

The first argument must be a type string identifying the event. The string can include a detach category. Additionally, if no event specifically matches the type string, a conditional event might be used if its test(..) method indicates a match. Otherwise, the default event definition will be used for the specified type.

Individual events can define how the subscription params are handled, but the default signature is after( type, callback, thisObj, arg0..argN ) where thisObj will be used for 'this' in the callback, and the additional arguments will be passed to the callback after the event object.

Parameters:
type <String> event type to subcribe to, with optional detach category
arg* <any> see above note on default signature
Returns: Object
this instance
Chainable: This method is chainable.

detach

Object detach ( typeSpec , phase , args* )

Unsubscribe one or multiple subscribers. Some example signatures are:

Called withWhat is detached
detach() All subscriptions to all events in all phases
detach(subscriptionObject) That subscription
detach("foo") All subscriptions to event "foo" in all phases
detach("foo", "before") All subscriptions to event "foo" in the "before" phases
detach("foo", "before", callbackFunc)* All subscriptions to event "foo" in the "before" phase that are bound to callbackFunc (*See below for notes)
detach("cat:*") All subscriptions to all events in all phases that were subscribed with category "cat" (e.g. on("cat:foo", ...))
detach("cat:foo") All subscriptions to event "foo" in all phases that were subscribed with category "cat"

Note, parameters beyond the type and phase are passed to the event's filterSubs method to apply any signature specific filtration for the subscriber list, so the detach signature that passes the callback is just an example of the signature supported by the default event.

Parameters:
typeSpec <Object|String> (optional) event subscription object or event type string, optionally with category
phase <String> (optional) phase from which to detach
args* <any> additional arguments used by the event's filterSubs method to better isolate which sub(s) to detach.
Returns: Object
this instance
Chainable: This method is chainable.

Returns: Array

eventDef.fire

protected static void eventDef.fire ( host , type , args* )

Executes subscribers for the "before" (aka "on") phase for all targets in the bubble chain until propagation is stopped or the last target is notified. If not prevented and if the event has one, the default behavior is executed. Then the executes subscribers for the "after" phase up the bubble chain.

If the event is prevented and it has one, the ifPrevented method is executed. "after" phase subscribers are not executed if the behavior is prevented.

Similarly, if the event propagation is stopped and it has one, the ifStopped method is executed. Note, this will not prevent the default behavior or the "after" subscribers from being executed.

Parameters:
host <Object> the instance from which fire was called
type <String> the event type to dispatch
args* <any> additional args passed fire(type, _here_...)

eventDef.generateEvent

protected static Object eventDef.generateEvent ( host , type , args* )

Called by fire. Creates the event object that will be passed to the subscribers. The event has the following properties and methods:

  • type
  • target
  • preventDefault()
  • _prevent (reserved)
  • stopPropagation()
  • stopImmediatePropagation()
  • _stop (reserved)
  • halt() - stopPropagation() + preventDefault()
  • detach() - detaches the current subscriber
  • details - array of additional args passed to fire(type, _here_...)

If the first additional argument is an object, its properties will be added to the event object directly as well as being the object being included in the details property.

Before it is passed to subscribers, its currentTarget and subscription properties are updated accordingly by the notify method.

Parameters:
host <Object> the instance from which on/after was called
type <String> the name of the event
args* <any> additional data provided to subscribers.
Returns: Object
the event object

eventDef.init

protected static void eventDef.init ( host , sub )
Initial one-time setup for an event. E.g. for DOM events, only one actual DOM subscription is made, the first time a subscription is requested. A custom event is used to manage this and subsequent subscriptions in order to guarantee subscriber execution order. Also see eventDef.destroy.
Parameters:
host <Object> the instance from which on/after was called
sub <Object> the initial subscription object

eventDef.notify

protected static void eventDef.notify ( path , event , phase )

Executes all the subscribers in a bubble chain for an event in a given phase ("before" or "after"). Used by fire.

If a subscriber calls e.stopImmediatePropagation(), no further subscribers will be executed, and if a subscriber calls e.stopPropagation(), no further bubble targets will be notified.

Parameters:
path <Array> bubble targets in the order they should be notified
event <Object> the event to pass to the subscribers
phase <String> the phase location of the subscribers

eventDef.subscribe

protected static boolean eventDef.subscribe ( host , sub , args* )

Subscription setup or other work that needs to happen each time a subscription is requested. At the point this method receives the subscription, it will only contain the following properties:

  • type - the event name
  • phase - "on" or "after" (unless your event supports more than these two default phases)
  • category - (string) if the subscription was generated with a detach category, e.g. for on('foo:click', ...), this would contain "foo"
  • host - the instance from which on(..) was called
  • detach() - method to detach this subscription

It is the purpose of this method to add anything uniquely appropriate for this event to the subscription, such as sub.callback to facilitate notifications or identifying it for detach.

Return true to allow the subscription to occur, or false to abort.

Parameters:
host <Object> the instance from which on/after was called
sub <Object> the initial subscription object
args* <any> any additional arguments passed on(type, _here_...)
Returns: boolean
true to allow subscription, false to abort

eventDef.test

protected static boolean eventDef.test ( host , type , args* )

Test the provided event type string and other args passed to on(type, ...) or after(type, ...) to indicate that this event definition should handle the subscription. By default, event mapping is a direct type string => event definition. This allows for more generic type strings that can be used to trigger specific behavior in the init or subscribe methods. An example would be a regex based test that matched on("key(shift+enter)", ...).

Return true to indicate that this event should handle the subscription.

This is optional for any event, but can also be included in a default event. By default, a simple type => event mapping is used.

Parameters:
host <Object> the instance from which on/after was called
type <String> the type string passed to on(..) or after(..)
args* <any> additional arguments after type and phase (e.g. callback function)
Returns: boolean
true to indicate that this event should handle the subscription

fire

Object fire ( type , args* )

Trigger the execution of subscribers to a specific event. The particular logic used for the notification is defined in the event definition's fire(..) method. If not specified there, the default event's fire(..)

As noted in eventDef.fire, the global default event's fire executes the "before" (aka "on") phase subscribers, then the event's default behavior if it has one, then the "after" phase subscribers, but this can be overridden per event. More detail is available in the API doc for the (protected) eventDef.fire method.

The type string alone will be used to identify the event whose fire definition should be used, and will not be compared against conditional events' test(..) methods.

Parameters:
type <String> the type identifying the event whose fire(..) to use.
args* <any> extra arguments to pass along
Returns: Object
this instance

publish

Object publish ( eventDef )

Add a new event to this instance's collection of events. Use this to add an event with specific default behavior, ifPrevented behavior, or special subscription/detach logic (etc). If the event doesn't behave in any way different from the default, you don't have to publish it. If the event applies to all instances, define it in the static _events collection for the class.

Accepts an event definition object with properties and methods to override those defined in the defaultEvent. See the description of Y.Event.API.defaultEvent for the properties and methods to include. Any methods or properties not defined will be provided by the defaultEvent.

If the pattern property is set to a regular expression, it will be wrapped in a memoized test function automatically.

Include a type property in the event definition, or pass the type string as the first parameter and the overrides as the second.

Parameters:
eventDef <Object> collection of event methods/properties
Returns: Object
the instance
Chainable: This method is chainable.

subscribe

Object subscribe ( type , phase , arg* )

Subscribe to an event on this object. This method is a catchall for events that might support more than the standard "before" (aka "on") and "after" phases. This method allows for subscription to any event phase.

The first argument must be a type string identifying the event. The string can include a detach category. Additionally, if no event specifically matches the type string, a conditional event might be used if its test(..) method indicates a match. Otherwise, the default event definition will be used for the specified type.

The second argument must be a phase string. Passing the string "before", for example, mimics the behavior of on(...).

Individual events can define how the subscription params are handled, but the default signature is subscribe( type, phase, callback, thisObj, arg0..argN ) where thisObj will be used for 'this' in the callback, and the additional arguments will be passed to the callback after the event object.

Parameters:
type <String> event type to subcribe to, with optional detach category
phase <String> event phase to attach subscription
arg* <any> see above note on default signature
Returns: Object
this instance
Chainable: This method is chainable.
